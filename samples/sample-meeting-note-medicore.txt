- 기업명: 주식회사 메디코어 (Medicore Inc.)
- 업종: 의료기기 제조업
- 직원 수: 총 148명 (정규직 130명, 비정규직 18명)

- 사업 내용: 체외진단 의료기기 개발·제조 및 진단키트 ODM 수출. 주력 제품은 혈당측정기, 감염병 신속진단키트, 바이오센서. 국내 병원·약국 유통과 동남아·중동 수출 병행.
- 고객 유형: B2B (병원·약국·해외 바이어)

2. AI 도입 현황

현재 AI 도입 단계는 '3단계 – 부분 도입' 수준이라 판단됨.
- 개발팀에서 GitHub Copilot을 6개월 전부터 사용 중 (5명)
- 품질관리팀에서 이미지 기반 불량 검출 파일럿을 작년 12월에 시도했으나, 정확도 부족(82%)으로 보류 상태
- 영업팀에서 ChatGPT Enterprise를 활용해 RFP 응답 초안 작성 (3개월 전 도입)
- AI 전담 인력: 1명 (개발팀 소속 데이터 엔지니어, 머신러닝 경험 있음)
- AI 도구 구독료: GitHub Copilot 5계정 + ChatGPT Enterprise 10계정 = 월 약 85만원
- AI 교육 예산: 연 500만원 (외부 세미나 참석 위주, 체계적 교육 프로그램은 없음)

핵심 문제: "데이터는 있지만 흩어져 있고, 쓸 수 있는 사람이 없다"
→ 대표이사 발언: "공장 데이터가 MES에, 고객 데이터가 CRM에, 영업 데이터가 Excel에 따로 있어서 통합 분석이 안 된다. AI를 도입하려면 데이터를 먼저 정리해야 하는데, 그걸 할 사람이 없다."

3. 5개 영역 자체 진단 점수

참석자 합의 기반 점수:
- AI 전략: 2.5 → CTO가 AI 도입 방향을 구상 중이나, 공식 전략/로드맵 문서 없음. 대표이사는 적극 지지하나 구체적 목표 미설정
- 데이터: 2.0 → MES, CRM, ERP 등 시스템별 데이터 존재하나 통합 안 됨. 데이터 품질 관리 프로세스 없음. 공장 센서 데이터 일부 수집 중이나 활용 안 됨
- 프로세스: 2.8 → 생산 공정은 MES로 어느 정도 자동화. 영업·CS·인사는 대부분 수동. RFP 응답에 ChatGPT 활용 시작
- 인재역량: 2.0 → 데이터 엔지니어 1명이 유일한 AI 역량. 개발자 15명 중 ML 경험 2명. 비개발 직원의 AI 리터러시 낮음
- 기술환경: 3.5 → AWS 클라우드 사용 (EC2, S3, RDS). 자체 개발팀 보유. GitHub 기반 개발 프로세스. 다만 MLOps 파이프라인 없음

4. 우선 과제 논의

미팅에서 집중 논의된 과제:

1순위: 품질검사 AI 자동화 (이미지 불량 검출 고도화)
- 현재 파일럿 정확도 82%를 95% 이상으로 끌어올려야 함
- 품질관리팀에서 매일 수작업으로 500건 이상 육안 검사 중
- 대응 모듈: [D] 해커톤 (빠른 PoC 개발)
- 긴급도: 높음 — FDA 인증 갱신 일정 상 올해 상반기 안에 성과 필요

2순위: 영업·수출 데이터 통합 대시보드 구축
- 현재 영업 데이터가 CRM, ERP, Excel에 분산
- 수출국별·제품별 매출 분석에 주 3일 이상 소요
- 대응 모듈: [B] 컨설팅
- 긴급도: 높음

3순위: 전사 AI 리터러시 교육 프로그램
- 비개발 직원 130명 중 AI 도구를 사용하는 사람이 13명(10%)에 불과
- 대응 모듈: [A] 워크숍
- 긴급도: 중간

권장 경로: [A] 워크숍 → [B] 컨설팅 → [D] 해커톤 → [C] 고급 교육 → [F] AI서비스

5. 부서별 Pain Point (현장 목소리)

[품질관리팀 – 팀장 발언]
"불량 검사를 눈으로 보고 판정하는데, 하루 500건 이상. 주당 25시간 이상을 검사에 쓰고 있다. 사람마다 판정 기준이 달라서 불량률 편차도 크다. AI 비전 검사 도입하면 확 줄일 수 있을 텐데."

[영업팀 – 팀장 발언]
"수출 바이어한테 보고서 보내려면 CRM이랑 ERP 데이터를 일일이 엑셀로 취합해야 한다. 국가별·분기별 데이터 정리하는 데만 주 15시간. 보고서 양식도 바이어마다 달라서 매번 수작업이다."

[개발팀 – 팀장 발언]
"신제품 개발 시 인허가 문서 작성이 가장 병목이다. KFDA·FDA 규격에 맞는 기술문서를 작성하는 데 건당 80시간 이상 걸린다. 비슷한 구조인데 매번 처음부터 쓴다. 주당으로 치면 약 20시간을 인허가 문서에 쓰고 있다."

[인사팀 – 팀장 발언]
"채용 공고 작성, 이력서 스크리닝, 교육 이수 관리를 다 수동으로 한다. 특히 이력서가 월 200건 이상 오는데 1차 스크리닝에 주 8시간. 기준도 담당자마다 달라서 일관성이 없다."

[재무이사 발언]
"월말 결산 리포트를 ERP에서 뽑아 엑셀로 가공하는 데 주 10시간. 경영진 리포트 양식이 자주 바뀌어서 자동화도 쉽지 않다. 원가 분석도 공정별 데이터를 MES에서 수동으로 뽑아야 해서 실시간 모니터링이 안 된다."

6. 내부 환경

[경영진 리더십]
대표이사 정민수 – "AI는 선택이 아닌 필수. 올해 AI에 최소 5천만원 이상 투자하겠다"는 의지 표명. 다만 구체적 목표나 KPI를 설정하지는 않은 상태. CTO 김태호는 기술적 이해도 높고, 파일럿 프로젝트를 직접 주도.

[조직 문화]
개발팀·품질관리팀은 새로운 도구에 개방적. 반면 영업팀·인사팀은 "현재 방식이 익숙하다"는 저항감 있음. 전반적으로 변화 수용성은 '보통' 수준.

[데이터 현황]
MES(생산 데이터), CRM(Salesforce - 고객·영업), ERP(더존 iCUBE - 재무·회계), 별도 LIMS(실험 데이터). 시스템 간 연동 없음. 공장 IoT 센서 데이터는 로컬 서버에 저장 중이나 분석 안 됨.

[IT 인프라]
AWS 사용 중 (월 약 300만원). 개발팀 15명, GitHub Enterprise 사용. CI/CD 파이프라인(Jenkins) 구축됨. 자체 API 서버 운영. 다만 ML 학습 환경은 없음 (GPU 서버 미보유).

[협업 도구]
Slack 사용 중 (유료 플랜). 문서는 Confluence + Google Drive 혼용. 프로젝트 관리는 Jira.

[재무 여력]
영업이익률 약 12%. AI 투자 예산으로 올해 5천만원 확보 가능. 정부 AI 바우처 신청 예정 (최대 4천만원 매칭).

10. 기타 참고사항

- 대표이사: "경쟁사 S사가 AI 검사를 도입한 후 불량률이 확 떨어졌다는 걸 보고 위기감을 느꼈다. 우리도 올해 안에 반드시 성과를 내야 한다."
- CTO: "MLOps 파이프라인이 없어서 모델을 만들어도 운영에 올리기 어렵다. 인프라부터 잡아야 한다."
- 품질관리팀장: "검사 데이터는 이미 10만건 이상 축적되어 있다. 라벨링만 하면 바로 학습 가능하다."
- 영업팀장: "Salesforce 데이터가 3년치 있는데, 분석 한 번 못 해봤다. 대시보드만 있어도 큰 도움이 될 것."
