개요

미팅 노트를 붙여넣으면 Claude API가 구조화된 데이터를 추출하고, 사람이 리뷰/수정한 뒤, 기존 report.html 템플릿에 값을 주입하여 PDF를 생성하는 Next.js 웹앱.

사용자: 조코딩 AX 파트너스 컨설턴트 + 클라이언트 기업 (공유 용도)
배포: 로컬 개발 우선 (추후 Vercel 배포 가능)

유저 플로우

STEP 1
미팅 노트 붙여넣기
POST /api/report/analyze
→
STEP 2
리뷰 & 수정
클라이언트 state
→
STEP 3
PDF 다운로드
POST /api/report/pdf

기술 스택

Next.js 16 (App Router) TypeScript Tailwind CSS v4 Claude API (@anthropic-ai/sdk) Puppeteer (PDF) React Context (상태관리)

페이지 구조 (3개)

/report 미팅 노트 입력
textarea (노트 붙여넣기) · "리포트 생성" 버튼 · 로딩 스피너 · 에러 메시지
/report/review 리뷰 & 수정
6개 섹션 폼 · 편집 가능한 input/textarea · null 필드 = 노란색 경고 · low confidence = 주황색 표시 · "리포트 확정" 버튼
/report/preview 미리보기 & PDF
template-engine으로 치환된 HTML → iframe srcdoc 렌더링 · "PDF 다운로드" 버튼 · "다시 수정" 버튼

API 엔드포인트 (2개)

POST /api/report/analyze
입력: { meetingNotes: string }
동작: Claude API (tool_use)로 미팅 노트 → 25개 필드 JSON
출력: { fields: {...}, metadata: { fieldsExtracted, fieldsMissing, lowConfidenceFields } }
POST /api/report/pdf
입력: { fields: {...} }
동작: report.html 플레이스홀더 치환 → Puppeteer PDF 렌더링
출력: application/pdf 바이너리

Claude API 프롬프트 전략

System Prompt: 역할 정의 + JSON 스키마 + 예시값 + 추출 규칙
추출 규칙:
  • 노트에 명시된 정보 → 그대로 추출
  • 노트에 없는 정보 → null
  • 애매한 정보 → 최선 추정 + lowConfidenceFields에 추가
  • 점수(1.0~5.0) → 노트 기반 판단, 근거 부족 시 low confidence
응답 형식: tool_use (structured output)로 JSON 보장

상태 관리

React Context (ReportContext)로 페이지 간 전달. DB 없음.
• meetingNotes: string — 원본 미팅 노트
• fields: ReportFields — 추출/수정된 필드
• metadata: ExtractMetadata — missing/low confidence 정보

에러 처리

Claude API 실패: 에러 메시지 + 재시도
PDF 생성 실패: 에러 메시지 + 재시도
빈 노트: 버튼 비활성화
필수 필드 누락: 경고 (생성은 허용)
